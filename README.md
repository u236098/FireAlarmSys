# Fire Alarm WSN

This folder contains the complete fire alarm project.

```text
Arduino UNO <-> UART <-> Zolertia sender <-> 802.15.4 wireless <-> Zolertia receiver/root <-> dashboard
```

The two Zolertia boards communicate wirelessly through Contiki-NG 802.15.4, 6LoWPAN, RPL, and UDP. The cable between the Arduino and the sender Zolertia is only the local UART sensor/actuator link. The receiver/root Zolertia is connected to the computer running the dashboard by USB serial.

Run commands from its folder eg:

```sh
cd "/home/kali/Desktop/contiki- communication/fire_alarm"
```

## Hardware Roles

Use the stable `/dev/serial/by-id` names. Do not rely on `/dev/ttyUSB0` or `/dev/ttyUSB1`, because they can swap after reconnecting USB cables.

```text
Arduino UNO:
  /dev/serial/by-id/usb-Arduino__www.arduino.cc__0043_7413633393235130B1D2-if00

Zolertia receiver/root, used by the dashboard:
  /dev/serial/by-id/usb-Silicon_Labs_Zolertia_RE-Mote_platform_ZOL-RM01-A000704-if00-port0

Zolertia sender, wired to the Arduino:
  /dev/serial/by-id/usb-Silicon_Labs_Zolertia_RE-Mote_platform_ZOL-RM01-A000729-if00-port0
```

Check connected devices:

```sh
make ports
```

## Wiring

Arduino to sender Zolertia:

```text
Arduino pin 10 RX  <-  Zolertia PC0 TX
Arduino pin 11 TX  ->  Zolertia PC1 RX
```

The prototype used a direct Arduino pin 11 to Zolertia PC1 connection. This worked during testing, but Arduino pin 11 outputs 5V logic and Zolertia PC1 is designed for 3.3V logic. A level shifter or voltage divider is recommended for a safer final revision.

## Run The Dashboard

With real hardware connected:

```sh
make dashboard
```

Without hardware, using simulated sensor data:

```sh
make dashboard-demo
```

Open:

```text
http://127.0.0.1:8080
```

The dashboard must read the receiver/root Zolertia. If the serial log shows `uart-client`, `Arduino line`, or `ACK from root`, you are reading the sender port. The correct receiver stream prints `DASHBOARD_JSON`.

Check the dashboard API:

```sh
make api
```

## Buzzer Control

The dashboard can send commands back to the Arduino:

```text
dashboard button
  -> receiver/root Zolertia over USB serial
  -> sender Zolertia over wireless UDP
  -> Arduino over UART
  -> buzzer pin 7
```

Dashboard buttons:

```text
Buzzer ON   manual override on
Buzzer OFF  manual override off
Auto        normal fire-alarm behavior
```

Command-line shortcuts:

```sh
make buzzer-on
make buzzer-off
make buzzer-auto
```

`Auto` returns the buzzer to normal fire-alarm behavior, where the Arduino turns it on only when temperature or gas is above the alert threshold.

Important: commands can only be delivered after the receiver/root has received at least one sensor packet from the sender. That first packet tells the receiver/root the sender's wireless IPv6 address.

## Flash Firmware

The Makefile uses `arduino-cli` to compile and upload the Arduino sketch. If `arduino-cli` is installed in a different location, override it when running `make`:

```sh
make upload-arduino ARDUINO_CLI=/path/to/arduino-cli
```

Flash all active devices:

```sh
make flash-all
```

Individual targets:

```sh
make flash-receiver
make flash-sender
make upload-arduino
```

Override ports if needed:

```sh
make dashboard RECEIVER_PORT=/dev/ttyUSB0
make flash-sender SENDER_PORT=/dev/ttyUSB1
make upload-arduino ARDUINO_PORT=/dev/ttyACM0
```

## Data And Live Plots

The dashboard shows live plots for:

```text
temperature
humidity
gas
alert and buzzer state
```

The plot area includes a timeline control. Leave `Live` enabled to follow the newest data, or disable it and move the slider to inspect older samples from the same dashboard run. The dashboard keeps about six hours of in-memory plot history at the current two-second sensing interval.

CSV logs are written to:

```text
data/fire_alarm_YYYYMMDD_HHMMSS.csv
```

These CSV files act as the simple project database. They store timestamps, packet IDs, sensor values, alert flags, buzzer state, and alert cause.

Remove generated build outputs, CSV logs, caches, and dashboard logs:

```sh
make clean-generated
```

Do not run `make clean-generated` before submission if you want to keep CSV logs as database evidence.

## Source Code Summary

The active code is:

```text
arduino/fire_alarm/fire_alarm.ino
  Arduino sensing, local alarm logic, LCD/fan/buzzer control, and UART command handling.

contiki/uart_udp_client/uart-udp-client.c
  Sender Zolertia firmware. Bridges Arduino UART data to wireless UDP and forwards dashboard commands back to Arduino.

contiki/udp_dashboard_receiver/udp-dashboard-receiver.c
  Receiver/root Zolertia firmware. Starts the RPL root, receives wireless packets, and exposes them to the dashboard over USB serial.

dashboard/fire_alarm_dashboard.py
  Python web dashboard. Shows live values, live timeline plots, CSV logging, serial log, and buzzer controls.
```
